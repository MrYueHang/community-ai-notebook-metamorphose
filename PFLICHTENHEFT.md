# PFLICHTENHEFT (System Specs) v2.0

## 1. System Overview
Gezypolis v2.0 ("Omni-Layer") transforms the spatial OS into a multi-departmental productivity tool.

## 2. Modules (The "Layers")

### 2.1 Project Management (PM)
*   **Source Data:** Abstracted from a "2000-page Fullstack PM Script".
*   **UI:** Kanban/List view with cost/time attributes.
*   **Accessibility:** "Boomer-Friendly" large text and clear status indicators.

### 2.2 Knowledge Base (Wiki)
*   **Structure:** Structured extraction of community rules and technical documentation.
*   **Access:** Dedicated "Wiki" room in the 3D environment.

### 2.3 Special Zones
*   **Court:** Legal logic implementation.
*   **Health:** System diagnostics.
*   **Chess:** Strategic simulation (Visual only in v2.0).

## 3. UX Guidelines
*   **Navigation:** Must use the "Omni-Deck" pattern (Large, centered buttons).
*   **Feedback:** Visual atmosphere changes (Lighting/Fog) upon navigation to reinforce context switching.
*   **AI:** LettaAI must be persistent and context-aware.

## 4. Technical Constraints
*   **React Version:** Must stay on React 18.x until R3F updates.
*   **Performance:** 3D assets must be procedural (Box/Sphere/Text) to ensure fast loading without external assets.
